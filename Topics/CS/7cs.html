High Level & Low Level Languages

Definition

Low level languages are machine code (binary values). They are also the language of an assembly (translate a higher level language into machine code). Low level languages are faster compared to high level languages because they don't have to be translated as much as high level languages

High level languages is programming language close to human language. They have statements that are easier to understand, this means a lot of people do not have to have high knowledge about machine code or hardware instructions to start codigng

Key Terms

  • Assembly Language - an assembly language is a special language made for a specific system/hardware. Few programmers choose to write programs in an assembly language as it is difficult and not universal.
  • Machine Code - machine code is written in binary. It is very hard for humans to write programs by inputting binary as this takes a lot of time and expertise.
  • Translators - Translates code - Compiler, Interpreter, Assembler

    Compiler Interpreter Assembler
    Translates a high-level language program into machine code. Executes a high-level language program a statement at a time. Translates a low-level language program into machine code.
    An executable file of machine code is produced No executable file of machine code is produced. An executable file of machine code is produced
    One high-level langauge statement can be translated into several machine code instructions One high-level language program statement may require several machine code instructions to be executed One low-level language statement is usually translated into one machine code instruction
    Compiled programs are used without the compiler Interpreted programs cannot be used without the interpreter Assembled programs can be used without the assembler
    A compiled program is usually distributed for general use. An interpreter is often used when a program is being developed An assembled program is usually distributed for general use.
